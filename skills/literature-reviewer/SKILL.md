# literature-reviewer — 参考文献の調査・平易な要約・URL検証

## 役割

参考文献まわりの3つの仕事を1回で行う：

1. **リスト作成**：文献を収集・特定し、書誌情報（著者・年・タイトル・venue）を確定する
2. **平易な概要**：各文献を**中学生でもわかるレベル**の言葉で、数行に要約する
3. **URL検証**：本文を取得できるURLを特定し、**確認できないものは ⚠️ 出典不明・要確認** とする

## 入力

- テーマ・キーワード（例：「時系列データの粒度変換」）
- 既存のドラフト・参考文献リスト・DOI／タイトル群
- ホストスキル（paper-compiler / research-proposer）からの呼び出し（素材・初期質問テーブルの内容を引き継ぐ）
- **literature-researcher からの候補リスト**（研究開始時の虎の巻）── researcher＝探索、本スキル＝検証・整形、の分業

未指定の場合は会話から読み取り、確定できない条件（分野・対象年代・件数の目安）のみ質問する。

## 実行手順

1. **収集・特定**：web検索で各文献の**実在を確認**し、著者・年・タイトル・venue を確定する。テーマ指定の場合は「7割・2020年以降」の法則（最新7割＋オーソリティ数本＋データ原典）を目安に構成する。
2. **エントリ生成**：文献ごとに次を作成する。
   - 書誌情報（venue 指定形式があればそれに従う。未指定は AMA 形式）
   - **わかりやすい概要（数行）**：専門用語を言い換え、「何を・どうやって・何がわかった」を中学生でもわかる言葉で。アブストラクトの直訳は禁止。
   - **本文URL**：次の優先順で特定する ── DOI（https://doi.org/…）→ 出版社ページ → PubMed / PMC → arXiv（**preprint と明記**）→ 著者公開版（大学リポジトリ等）。
3. **URL検証**：候補URLに実際にアクセスして到達性を確認する。確認できない場合は **⚠️ 出典不明・要確認** とし、判明している範囲の書誌のみを記載して「何を確認すべきか」（例：DOIの再確認、図書館経由の入手）を添える。
4. **品質レポート**：リスト全体について、2020年以降の構成比（7割目安。箱テンプレ骨格では N/A）、arXiv の preprint 明記の有無、⚠️ の件数を報告する。

## 出力フォーマット

```
## 📚 Literature Review（関連研究）

| # | 文献（著者・年・タイトル・venue） | わかりやすい概要 | 本文URL | 判定 |
|---|---|---|---|---|
| 1 | 〔書誌〕 | 〔中学生でもわかる数行〕 | https://doi.org/… | ✅ |
| 2 | 〔書誌（判明分のみ）〕 | 〔同上〕 | ─ | ⚠️ 出典不明・要確認（〔確認方法〕） |

**品質レポート**：全N件 ／ 2020年以降 X%（目安70%） ／ arXiv preprint明記 ✅ ／ ⚠️ M件
```

判定記号：✅＝本文URLの到達を確認済み ／ ⚠️＝出典不明・要確認（URL未確認・実在未確認・アクセス不能）

## ホスト文書への挿入規定（改ページ）

呼び出し元に応じて、生成したセクションを**改ページで**該当位置に挿入・更新する。DOCX の編集は docx スキルの手順（unpack → XML → pack）に従い、`validate.py` で検証する。

| 呼び出し元 | 挿入位置 |
|---|---|
| **paper-compiler**（paper_template_vX.docx） | **「初期質問テーブル」と「論文ドラフト領域」の間**。文書は ①初期質問テーブル →改ページ→ ②Literature Review（関連研究）→改ページ→ ③論文ドラフト領域 →改ページ→ ④レビュー履歴（最後尾） の4部構成になる |
| **research-proposer**（Proposal_vX.docx） | 「投稿先・資料・リンク」の後・**「FAQ・想定問答」の前**に配置する（文書の最後尾は「更新履歴」） |

- 再コンパイル／改稿のたびに同セクションを**上書き更新**する（版管理はホスト文書の vX に従う）。
- **更新箇所にはWordコメントを必ず付す**：追加・削除・判定が変わった文献の行に、`[vX 更新] 文献3件を追加（2020年以降）／理由：⚠️出典不明だった2件をDOI確認により✅へ` のように「何を・なぜ」を1〜3行で残す。著者名は必ず `Yusuke Takahashi, PhD`（initials `YT`）とする（Claude名義にしない）。前版のコメントは削除しない。
- **投稿版（camera-ready）**では、✅の文献を本文の References へ転記したうえで本セクションを削除する（入り切らない分は Supplementary References へ）。
- ⚠️ の文献は、解消されるまで本文の References に**転記しない**（audit 項目10・11 とも連動）。

## 禁止事項（最重要）

1. **でっち上げ禁止**：実在が確認できない文献・DOI・URLを生成しない。もっともらしい書誌を創作することは本スキルの最大の違反。確認できなければ必ず ⚠️。
2. ⚠️ を出すときは必ず「何を確認すべきか」を添える（指摘だけは禁止）。
3. 概要はアブストラクトの直訳・コピーをしない。必ず平易に言い換える（著作権と読みやすさの両面から）。
4. arXiv 文献は必ず preprint と明記する（AI / ML 分野は明記のうえ引用可）。
